# Commit Guidelines

Keep commits small, focused, and reversible. Use these simple rules when contributing:

- Use the imperative, present tense in commit messages (e.g., "Fix typo", "Add tests").
- Start the subject line with a short type (e.g., "Feat:", "Fix:", "Tidy:").
- Keep the subject line under 72 characters.
- Write a short body if more context is needed.
- Make one logical change per commit.

Example:

Feat: Add export to CSV for login history

This adds CSV export for the login history page, including headers and timestamps.
